# Phase Sequencing

## Purpose
Order work so planning, implementation, validation, commit, and push do not blur together.

## Use When
A task spans multiple decisions or milestones.

## Practice
- Inspect first.
- Define milestone and boundaries.
- Edit within approved scope.
- Validate before commit readiness.
- Commit only after approval.
- Push only after separate approval.

## Anti-Patterns
- Starting implementation before boundary review.
- Combining commit and push approval.
- Skipping closeout because the code appears done.

## Validation Or Report Expectations
- Report current phase, next gate, required approval, and validation status.
