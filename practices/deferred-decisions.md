# Deferred Decisions

## Purpose
Keep unresolved choices visible without forcing premature action.

## Use When
A decision affects scope, architecture, runtime behavior, publishing, or branch policy.

## Practice
- Name the decision.
- Explain why it is deferred.
- State what evidence or approval is needed.
- Keep work moving only in areas that do not depend on the decision.

## Anti-Patterns
- Treating silence as approval.
- Encoding deferred choices as hidden defaults.
- Blocking unrelated safe work.

## Validation Or Report Expectations
- Report deferred decisions, owner input needed, and the next safe action that does not cross the deferred boundary.
