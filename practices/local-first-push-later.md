# Local First, Push Later

## Purpose
Separate local milestone quality from public publishing.

## Use When
Work should be committed locally before it is visible remotely.

## Practice
- Validate locally.
- Commit only after owner approval.
- Re-check status, branch, remote, and latest commits before push.
- Push only after separate owner approval.

## Anti-Patterns
- Auto-pushing after commit.
- Treating "safe to push" as approval.
- Publishing dirty, unreviewed, or ambiguous work.

## Validation Or Report Expectations
- Report local latest commit, remote target, clean status, and explicit push approval status.
