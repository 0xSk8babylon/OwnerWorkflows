# Implementation Change PR

## Purpose
Describe app or service behavior changes.

## Use When
Opening a PR that changes runtime code, API behavior, persistence, UI behavior, or tests.

## Scope
Implementation changes approved for the branch.

## Commands / Checklist
- Summary:
- User-visible behavior:
- Files changed:
- API/schema/data impact:
- Compatibility notes:
- Tests/builds run:
- Rollback or follow-up notes:

## Stop / Report Conditions
- Scope expanded beyond approval.
- Schema, API, security, or data changes lack owner approval.
- Required validation failed or did not run.

## Constraints
- Do not hide runtime impact.
- Do not include secrets.
- Do not merge without passing required checks.
