# Docs Change PR

## Purpose
Describe documentation-only changes.

## Use When
Opening a PR that changes docs, process notes, or governance text only.

## Scope
Documentation and repo notes.

## Commands / Checklist
- Summary:
- Changed docs:
- Runtime impact: none
- Validation: `git diff --check`
- Owner approval needed for doctrine, publishing, or policy changes:

## Stop / Report Conditions
- App code, schema, API, data, or CI files changed.
- Documentation implies behavior not implemented.

## Constraints
- Do not claim runtime behavior changed.
- Do not include secrets or private project details.
- Do not merge without required owner review.
