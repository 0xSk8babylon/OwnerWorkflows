# Stabilization Change PR

## Purpose
Describe fixes that reduce risk without broad feature work.

## Use When
Opening a PR for bug fixes, validation fixes, branch cleanup, or workflow stabilization.

## Scope
Narrow stabilization work only.

## Commands / Checklist
- Summary:
- Problem stabilized:
- Files changed:
- Behavior changed:
- Validation run:
- Remaining risks:
- Follow-up deferred:

## Stop / Report Conditions
- Fix becomes a feature.
- Validation requires unapproved tooling or dependency changes.
- Branch cleanup would delete or rewrite history.

## Constraints
- Do not force push.
- Do not rewrite history.
- Keep changes scoped to the stated stabilization target.
